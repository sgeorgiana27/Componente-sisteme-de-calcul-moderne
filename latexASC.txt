\subsection{Definiții}
\subsubsection{Definirea unui sistem de calcul}
\textcolor[RGB]{124, 13, 208}{Sistemul de calcul este un ansamblu de componente interconectate destinat prelucrării datelor, unde componentele sunt resurse ce contribuie la funcționarea sistemului.}

\subsubsection{ \textcolor[RGB]{147,30,116}{Resurse}}

\begin{enumerate}
    \item \textcolor[RGB]{147,30,116}{fizice} (componenta hardware)
    \begin{itemize}
    \item structura unui sistem de calcul minimal
    \item structura unui sistem de calcul elementar  (uzual)
    \item structura unui sistem de calcul specializat
    \end{itemize}
\item \textcolor[RGB]{147,30,116}{logice} (subsistemul software)
\begin{itemize}
    \item aplicații de sistem
    \item aplicații utilizator
\end{itemize}
\item \textcolor[RGB]{147,30,116}{informaționale}\textcolor[RGB]{124, 13, 208} {
-datele, fișierele de date ale utilizatorilor.}


\textcolor[RGB]{124, 13, 208} {Această clasificare suprapune termenii de informație și date.}

\textcolor[RGB]{124, 13, 208} {Informația este dată prelucrată.}

\end{enumerate}

\subsubsection{ \textcolor[RGB]{147,30,116}{Abordarea SC la nivel hardware}}
\hspace{2cm}


\begin{itemize}
\item \textcolor[RGB]{147,30,116}{organizarea componentelor} – pt. PERFORMANŢĂ  
\begin{enumerate}
\item cum operează fiecare componentă hard
\item posibilităţi de interconectare 
\end{enumerate}
\item \textcolor[RGB]{147,30,116}{proiectare (design)} - pt. UTILIZATOR \textcolor[RGB]{124, 13, 208}{(componentele sunt văzute în ansamblul lor. La acest nivel se aleg acele componente care răspund optim cerințelor utilizatorului)}
\begin{enumerate}
\item se aleg componentele hard
\item se aleg soluţiile de interconectare
\end{enumerate}
\item \textcolor[RGB]{147,30,116}{arhitectura sistemului de calcul}
\end{itemize}


\subsubsection{ \textcolor[RGB]{147,30,116}{Definirea arhitecturii calculatorului}}
\hspace{2cm}

\textcolor[RGB]{147,30,116}{Arhitectura sistemului de calcul se referă la structura şi comportamentul SC văzut de utilizator.}

Când ne referim la arhitectura unui SC trebuie să dăm
specificaţiile diferitelor unităţi funcţionale şi conectările fizice şi logice dintre ele.

Arhitectura calculatoarelor se referă la asamblarea subsistemelor
funcţionale într-un calculator numeric, pentru a obţine
performanţele impuse în condiţii de cost avantajoase. [DEX]

[Tanenbaum]
Sintagma arhitectura calculatoarelor este adesea utilizată cu
semnificaţia simplă de organizarea şi proiectarea calculatoarelor.
\textcolor[RGB]{147,30,116}{Arhitectura calculatoarelor se referă la studiul proiectării acelor părţi ale unui sistem de calcul care sunt vizibile programatorului.}

\textcolor[RGB]{147,30,116}{Nivelul de arhitectură} este legătura dintre subsistemele sitemului gazdă.


\textcolor[RGB]{147,30,116}{Arhitectura de nivel} este interfaţa dintre modulele funcţionale pe care le uneşte.

\begin{itemize}
\item \textcolor[RGB]{124, 13, 208}{AC = organizarea și proiectarea calculatoarelor, studiul proiectării acelor părți ale unui sistem de calcul
\item Controller = microprocesor care se ocupă cu prelucrarea informației}

\item \textcolor[RGB]{124, 13, 208}{UAL = unitate aritmeticologică}

\begin{itemize}
\item \textcolor[RGB]{124, 13, 208}{Limbaje funcționale: pascal,ocamel,sharp}
\item \textcolor[RGB]{124, 13, 208}{Limbaje logice: prolog}
\item \textcolor[RGB]{124, 13, 208}{Limbaje orientate pe aplicații web: html}
\end{itemize}
\item \textcolor[RGB]{124, 13, 208}{UAL e singura componentă a calculatorului care execută.}

\item \textcolor[RGB]{124, 13, 208}{ISA = industrial standard architecture = instruction set architecture}

\item \textcolor[RGB]{124, 13, 208}{PnP = plug and play (nu e necesar să instalăm noi o componentă, ea e “instalată” automat in momentul in care se conectează - sistemul de operare o detectează pe port, caută printe propriile programe un driver, dacă nu găsește și are conexiune caută online) $\leftarrow$ în cazul compilatoarelor nu atât de performante}

\item \textcolor[RGB]{124, 13, 208}{Manual de asamblare = specificațiile fiecărei componente pentru ca ea să fie folosită și asamblată în calculator}

\end{itemize}

\subsection{Structura ierarhică a unui sistem de calcul}


\textbf{Nivele conceptuale}
\begin{itemize}
\item \textcolor[RGB]{147,30,116}{N. logic-digital}

Porţi logice (tranzistoare → poartă logică → bistabil → registru)
\item \textcolor[RGB]{147,30,116}{N. microarhitecturii}

SC = regiştri + datapath + UAL
\item \textcolor[RGB]{147,30,116}{N. arhitecturii setului de instrucţiuni} (ISA)

Componentă → manual de utilizare → limbaj maşină
\item \textcolor[RGB]{147,30,116}{N. maşină al sistemului de operare}
\item \textcolor[RGB]{147,30,116}{N. limbajului de asamblare}
\item \textcolor[RGB]{147,30,116}{N. limbajului orientat pe problemă}
\end{itemize}

\subsection{Structura funcțională a SC}

\textbf{Structura fizică}

\textcolor[RGB]{124, 13, 208}{Structura fizică include enumerarea unităților funcționale}
\begin{itemize}
\item \textcolor[RGB]{147,30,116}{unităţi de schimb} – UCC \textcolor[RGB]{124, 13, 208}{- este formată din unități de schimb (este unitatea de comandă și control și cuprinde acele date care sunt implicate în transferul informației)}
\item \textcolor[RGB]{147,30,116}{componente} - UAL \textcolor[RGB]{124, 13, 208}{- unitatea aritmetologică și este formată din componente implicate direct în execuția operațiilor}
\item \textcolor[RGB]{147,30,116}{memorii interne} - regiştri + memorii;
\textcolor[RGB]{124, 13, 208}{Memoria (componente poziționate în interiorul box-ului calculatorului -
memorii interne, memorie implicată direct in execuția programului);}
\textcolor[RGB]{124, 13, 208}{
Regiștrii - registre, memorie realizată pe tehnologie RAM}
\item \textcolor[RGB]{147,30,116}{procesoare} - UCP
\textcolor[RGB]{124, 13, 208}{(unitatea centrală de procesare) este totalitatea procesoarelor
(unitățile de executare ale programelor)
}
\item \textcolor[RGB]{147,30,116}{dispozitive periferice} - IOS; \textcolor[RGB]{124, 13, 208}{Sistemul de intrare/ieșire este unitatea funcțională care grupează dispozitivele periferice (IOS = input/output sistem) }\textcolor[RGB]{124, 13, 208}{ - precizarea relațiilor dintre unitățile funcționale}
\end{itemize}

\textbf{Structura logică}

UCC+UAL=UCtrlP
      
UCtrlP+Registrii=UCP
       
UCP+Memorii=UC
            
UC+IOS=SC

\textcolor[RGB]{124, 13, 208}{UCtrlP= unitatea de control și prelucrare}


\textcolor[RGB]{124, 13, 208}{UC = unitatea centrală (box-ul calculatorului)}

\textcolor[RGB]{124, 13, 208}{SC = sistem de calcul}


\textcolor[RGB]{124, 13, 208}{UCP = unitatea centrală de procesare}


\subsection{Principiile von Neumann}

\begin{itemize}
\item Deoarece maşina este destinată calculelor, ea execută cele patru operaţii aritmetice. Se impune deci, existenţa unei componente specializate pentru calcule. (Astăzi, aceasta este numită \textcolor[RGB]{147,30,116}{unitatea aritmetico-logică }\textcolor[RGB]{124, 13, 208}{(UAL)}. )

\item Operaţiile se vor executa secvenţial. Pentru a asigura elasticitatea şi o relativă universalitate, se impune o distincţie între instrucţiunile necesare rezolvării unei probleme particulare şi controlul general asupra unor instrucţiuni executate la un 
moment dat. (Apare, astfel, ceea ce astăzi numim \textcolor[RGB]{147,30,116}{componentă de control }\textcolor[RGB]{124, 13, 208}{(UCC)}. ) 

\item Este necesară o componentă numită \textcolor[RGB]{147,30,116}{memorie internă }\textcolor[RGB]{124, 13, 208}{(MC)} în care „se ţin minte” pentru un timp limitat atât instrucţiunile, cât şi datele necesare rezolvării problemei. 

\item Cele trei componente, luate împreună, sunt analoage neuronilor din sistemul nervos central.

\item Se impune, continuând analogia, să existe un senzor care să recepţioneze semnalele de orice fel provenite din exterior. Este necesar, de asemenea, un motor care să acţioneze asupra mediului exterior. (Astăzi, acestea se numesc \textcolor[RGB]{147,30,116}{dispozitive de intrare/ieşire }\textcolor[RGB]{124, 13, 208}{(I/O)}. )

\item Este necesară o componentă care să „ţină minte permanent”, să înregistreze aşa cum omul îşi notează ceea ce nu doreşte să uite. Această componentă o vom numi \textcolor[RGB]{147,30,116}{memorie permanentă}. (Astăzi, ne referim la această componentă prin \textcolor[RGB]{147,30,116}{memorie externă}.) 

\item Este necesară crearea posibilităţii de trecere de la memoria internă la cea permanentă şi reciproc.
\end{itemize}

\subsection{Definire}

\textcolor[RGB]{124, 13, 208} {Sistemul de intrare/ieșire reprezintă unitatea funcțională formată din dispozitivele periferice, adică acele componente care trebuie să asigure pe de-o parte schimbul de informații și pe de altă parte organizarea memoriei pentru informațiile externe.}
\\
\\
\textcolor[RGB]{124, 13, 208}{
 Acestea pot să faciliteze un schimb de informații între memoria internă și mediul extern sau organizeze și să memoreze informații externe}
\textcolor[RGB]{124, 13, 208}{Cele ce facilitează un schimb de informații pot fi:}
\begin{itemize}
\item \textcolor[RGB]{124, 13, 208}{Periferice de sine stătătoare: tastatura, mouse, monitor}
\item \textcolor[RGB]{124, 13, 208}{Plăci de conexiune: ceea ce gândim ca structură la o placă de sunet, o placă de rețea, (și placa de bază intră în această categorie, dar trebuie totuși separată)}
\end{itemize}

\textcolor[RGB]{124, 13, 208}{Legat de organizarea și memorarea informațiilor externe:}

\begin{itemize}
\item \textcolor[RGB]{124, 13, 208}{Placa grafică = infrastructura suport pentru grafica de care dispune SO}
\item \textcolor[RGB]{124, 13, 208}{Procesor grafic = partea de prelucrare a parților grafice/video}
\end{itemize}


\textcolor[RGB]{124, 13, 208}{Numim periferic orice componentă care se conectează din exterior, pentru a crește funcționalitatea calculatorului.}

\subsection{Clasificări}
\begin{enumerate}
    \item \textcolor[RGB]{147,30,116}{din punct de vedere funcțional}
    \begin{itemize}
        \item periferice de schimb
        \item periferice purtătoare de informații\textcolor[RGB]{124, 13, 208}{(face diferența dintre o memorie externă - HDD, sau o memorie internă - RAM, care nu are proprietatea de a păstra informațiile permanent)}
    \end{itemize}
    \item \textcolor[RGB]{147,30,116}{după direcția de transfer a datelor \textcolor[RGB]{124, 13, 208}(în raport cu calculatorul)}
    \begin{itemize}
    \item periferice de intrare
    \item periferice de ieșire
    \item periferice de intrare/ieșire
    \end{itemize}
    \item \textcolor[RGB]{147,30,116}{după tipul de transfer al informației}
    
\begin{itemize}
\item periferice bloc (bloc adresabil = informații + control)
 \begin{itemize}
\item \textcolor[RGB]{124, 13, 208}{Transferă datele în "blocuri" de octeți, adică mai mulți octeți simultan (în același bloc). Avantajul transferului bloc este că fiecare astfel de unitate de transfer este adresabilă.(Spunem despre o entitate că are o adresă când putem să o identificăm in mod unic;
adresa = identificator unic).
}
\item \textcolor[RGB]{124, 13, 208}{Conținutul unui astfel de bloc e format din 2 tipuri de informații:}
\begin{itemize}
\item \textcolor[RGB]{124, 13, 208}{Informații propriu-zise (ex. Informațiile propriu-zise ale unui CNP sunt sexul, data nașterii și codul de oraș natal).}
\item \textcolor[RGB]{124, 13, 208}{Informații de control( ex. Informația de control a unui CNP este ultima cifră). Pe un HDD pista 0 are informații de control pentru toată disk-eta.}
\end{itemize}
\end{itemize}
        
\item periferice caracter (flux de octeți, neadresabil)


\textcolor[RGB]{124, 13, 208}{Este sub formă de transfer octet după octet (flux de octeți), într-o structură neadresabilă}

\textcolor[RGB]{124, 13, 208}{Email-ul are o trimitere pe bază de flux de octeți}

        
\end{itemize}
 \item \textcolor[RGB]{147,30,116}{după partenerul implicat în utilizare}
    \begin{itemize}
        \item periferice care interacționează cu omul
        \item periferice care interacționează cu mașina

\textcolor[RGB]{124, 13, 208}{Periferice care interacționează cu mașina (legătura între 2 componente fizice): modem (modulator de semnal), plăcile de conexiune, placa de bază, CDRom (unitatea de CD/DVD), rack-ul ca suport pt harddisk-uri fizice.}

   
\textcolor[RGB]{124, 13, 208}{În toate ierarhiile funcționează acest principiu.}

    \end{itemize}
\end{enumerate}

\subsection{Canal de intrare/ieșire}

\begin{itemize}
\item \textcolor[RGB]{147,30,116}{I/O processor} = unitate de prelucrare specializată 
pe operaţii I/O

\textcolor[RGB]{124, 13, 208}{Pot să văd un procesor ca o componentă intermediară între procesorul central și toate perifericele,
în așa fel încât această componentă să preia semnalele, să le prelucreze într-o măsura proprie și să transmită procesorului central doar ce are nevoie.}


\item realizează schimbul de date între memorie şi 
periferice
\item poate funcţiona în paralel cu procesorul central după numărul de periferice conectate.

\item Clasificare după numărul de periferice conectate:
\begin{itemize}
\item \textcolor[RGB]{147,30,116}{canal selector} – lucrează cu un singur periferic la un moment dat
\item \textcolor[RGB]{147,30,116}{canal multiplexor} – lucrează simultan cu mai 
multe periferice; pentru fiecare octet schimbat, trebuie să decidă care este destinaţia acestuia
\end{itemize}
\end{itemize}

\subsection{Legarea perifericelor la SC. Unităţile de interfaţă de I/O}
\subsubsection  {\textcolor[RGB]{147,30,116}{Interfaţă I/O}}

\begin{itemize}
    \item componentă hardware de legătură
    \item constă în linii speciale de comunicare care leagă perifericele la UCP
    \item  \textcolor[RGB]{124, 13, 208} {ea trebuie să asigure compatibilitatea componentelor}
\end{itemize}
\subsubsection{\textcolor[RGB]{147,30,116}{Funcțiile interfețelor I/O}}
\begin{itemize}
    \item supervizarea şi sincronizarea tuturor transferurilor I/O
    \item conversia semnalelor specifice modului de operare pe suporturi externe în semnale de lucru pe UCP
    \item sincronizarea ratelor de transfer de date de la viteza de lucru pe periferic la viteza de lucru în UCP
    \item decodificarea/codificarea datelor de pe periferic în format specific în UCP
    \item delimitarea semnalelor unui periferic de ale altora dintre cele conectate la acelaşi SC
\end{itemize}

\textcolor[RGB]{147,30,116}{\scalebox{1.5}{$\star$}}
\textcolor[RGB]{147,30,116}{Aceste funcţii trebuie integrate în contextul conexiunilor de tip: periferic – interfaţă – magistrală! }\textcolor[RGB]{147,30,116}{\scalebox{1.5}{$\star$}}

\subsubsection {\textcolor[RGB]{147,30,116} {Clasificarea interfețelor I/O}}
\begin{enumerate}
    \item \textcolor[RGB]{147,30,116} {după numărul semnalelor şi sensul de transmitere simultană}
    
    \begin{itemize}
        \item interfeţe seriale – transferă informaţia bi-direcţional
        \item interfeţe paralele – transferă informaţia numai într-un sens
    \end{itemize}
     \item \textcolor[RGB]{147,30,116}
     {după tipul de transmitere a datelor}
     \begin{itemize}
         \item interfeţe cu transmitere sincronă de date - viteza de transmitere a datelor depinde de frecvenţa de tact
         \item interfeţe cu transmitere asincronă de date - comunicarea se bazează pe structura datelor transmise (mesaj, pachet)
     \end{itemize}
\end{enumerate}

 \textcolor[RGB]{147,30,116}{I/O Bus} = magistrala de conectare a perifericelor în SC;
\\
\\
 Comunicarea între procesorul central şi periferice este permanentă:
 \begin{itemize}
     \item prin polling
     \item prin întreruperi (mai ales în configuraţiile time-critical)
 \end{itemize}

\subsection{Transferul eficient de date}

După gradul de implicare a procesorului în controlul
transferului de date, avem:
\begin{itemize}

 \item \textcolor[RGB]{147,30,116}{transfer prin program} – UCP se implică total în efectuarea transferului;
 
\item \textcolor[RGB]{147,30,116}{transfer prin întreruperi} – în care UCP poate să folosească timpul liber dintre două transferuri consecutive pentru executarea altor secvenţe de program (execuţie concurentă);

\item \textcolor[RGB]{147,30,116}{transfer prin acces direct la memorie} (DMA, engl. Direct Memory Access), fără intervenţia efectivă a procesorului; această metodă de transfer necesită un circuit specializat (numit controller DMA) care are posibilitatea de a prelua controlul magistralei sistem în vederea efectuării transferului propriu-zis de date;

\item \textcolor[RGB]{147,30,116}{transfer prin procesor de intrare/ieşire} – care conferă procesorului cel mai mare grad de libertate: sarcinile legate de transfer sunt preluate de către o altă unitate de tip procesor, numită procesor de intrare / ieşire.

\end{itemize}
$\star$
IOP (procesor sau canal I/O) apare ca un processor cu suport DMA pentru comunicarea cu perifericele; ÎN TIMPUL ÎN CARE procesorul central rezolvă task-urile de DMA, IOP poate încărca și executa propriile task-uri legate de operațiile de I/O.$\star$

\subsection {Placa de bază}


\begin{itemize}[label={\textcolor[RGB]{147,30,116}{$\bullet$}}]
    \item este principala placă de interconectare (circuite integrate) a calculatorului.
    \item din punct de vedere fizic, direct sau indirect, \textcolor[RGB]{147,30,116}{prin chipset} conectează procesorul cu toate componentele fizice din sistem.
\end{itemize}


 Diferențele semnificative de performanță între mai multe SC sunt generate de procesorele sistemelor respective, \textcolor[RGB]{147,30,116}{\textbf{dar}, fără un chipset și o placă de bază alese \textbf{potrivit}}, procesorul poate intra într-o stare de interblocare la nivelul datelor (data starvation, deadlock) față de celelalte componente.


 \textcolor[RGB]{147,30,116}{Chipset-ul și placa de bază} sunt \textbf{esențiale} pentru performanța SC și sunt  \textcolor[RGB]{147,30,116}{\textbf{singurele} componente} proiectate specific / orientat pe o arhitectura de procesor / familie de procesoare.


 Toate celelalte componente sunt proiectate independent de procesor, atât timp cât ele comunică \textcolor[RGB]{147,30,116}{prin magistrale \textbf{standard} recunoscute de chipset și de placa de bază}.


 Pentru a asigura compatibilitatea cu standardele mai vechi (legacy standards), cum sunt cele pe porturile seriale și paralele pentru tastatură, de exemplu, s-a adăugat un modul chip separat, numit \textcolor[RGB]{147,30,116}{super I/O chip}.

\subsubsection{\textcolor[RGB]{147,30,116}{Din punct de vedere funcțional}}
\begin{itemize}
 \item existența pe placa de bază a mai multor \textcolor[RGB]{147,30,116}{module chip} ( Northbridge, Southbridge și \textcolor[RGB]{147,30,116}{super I/O chips}) asigură un nivel
important de \textbf{flexibilitate} pentru arhitectura SC.


\item diferite Northbridge-uri cu designuri diferite, pot asigura \textcolor[RGB]{147,30,116}{compatibilitatea} procesorului cu diferite tipuri de
adaptoare grafice sau sloturi de memorie.


\item fiecare Northbridge poate fi
\textcolor[RGB]{147,30,116}{compatibil} cu mai multe chipuri de tip Southbridge, asigurând și mai multe \textcolor[RGB]{147,30,116}{combinații}.


\item toate aceste combinații pot folosi același
design de \textcolor[RGB]{147,30,116}{super I/O} pentru legarea
componentelor tradiționale.

\end{itemize}

\subsubsection{\textcolor[RGB]{147,30,116}{Din punct de vedere fizic}}

\hspace{0.45 cm}
Placa de bază implementează \textcolor[RGB]{147,30,116}{magistralele} care „țin” componentele conectate (procesor, chipset, etc) și asigură \textcolor[RGB]{147,30,116}{toți conectorii fizici} cu/pentru lumea exterioară.

\textcolor[RGB]{147,30,116}{Chipset-ul} constă dintr-o pereche de module chip responsabile de \textcolor[RGB]{147,30,116}{comunicarea} între procesor și celelalte componente pe placa de bază. 

Concret, unul dintre chip-uri, NorthBridge, răspunde de comunicarea cu componentele rapide și celălalt, SouthBridge, de comunicarea cu componentele lente. 

\textcolor[RGB]{147,30,116}{Arhitectura chipset-ului} este cea mai importantă alegere în proiectarea unei plăci de bază. Aceasta determină \textcolor[RGB]{147,30,116}{interfețele de magistrală} disponibile și, \textbf{mai departe}, tipurile de procesor, memorie principală, carduri / adaptoare grafice, alte memorii, plăci de extensie sau alte periferice pe care să le susțină (suporte) placa de bază.

Pentru fiecare modul și/sau chip de pe placa de bază, se ia o decizie specifică legată de amplasarea lui \textcolor[RGB]{147,30,116}{direct pe placă} \textbf{sau} prin intermediul unui \textcolor[RGB]{147,30,116}{soclu (socket)} în care să se conecteze (to be plugged into). Soclurile sunt mai scumpe, dar lasă deschisă posibilitatea de a schimba \textbf{sau} de a extinde (upgrade) ulterior chip-ul.

\subsection{Magistrale. Standarede de magistrală}

Un \textcolor[RGB]{147,30,116}{ chipset} se referă la un grup de circuite integrate, sau cipuri, fabricate pentru a lucra împreună. În cele mai multe cazuri, acestea sunt fabricate într-un singur produs.
În industria calculatoarelor, termenul de chipset este comun pentru a referi un set specializat de cipuri pe microprocesorul unui calculator și o pereche specifică de cipuri pe placa de bază.

Există două categorii de chipset-uri:
\begin{itemize}
    \item \textcolor[RGB]{147,30,116}{Chipset grafic}. Este componenta principală a plăcii video, coordonând activitatea plăcii. Un chipset grafic este caracterizat de frecvența de funcționare, lățimea bus-ului de transfer către memoria grafică și abilitatea acestuia de a executa instrucțiuni specifice. Principalii producători de chipset-uri grafice pentru PC-uri includ NVIDIA, AMD(prin achiziționarea firmei ATi), Intel etc.
    \item \textcolor[RGB]{147,30,116}{Chipset-ul plăcii de bază}. Pentru placa de bază, chipset-ul reprezintă componenta principală, influențând foarte mult performanța plăcii de bază. Este coordonatorul componentelor esențiale din sistem, prin urmare, performanțele chipset-ului plăcii de bază determină performanța configurației. Are două componente principale, northbridge și respectiv southbridge, ambele îndeplinind funcții distincte.
\end{itemize}

In alegerea unui chipset, în afară de indicii de performanţă evidenţi cu care se prezintă fiecare, mai trebuie ţinut cont şi de facilităţile suplimentare oferite, de exemplu video sau sunet integrat, caz în care nu mai aveţi nevoie de componente separate pentru subsistemul video sau audio. Totuși, de reținut că soluțiile "integrate" în chipset, nu vor egala ca performanță componentele separate, "dedicate".

    \item share-ability = facilitarea interfațării

\subsubsection{ \textcolor[RGB]{147,30,116}{Clasificarea după tipul componentelor interconectate}}
\begin{enumerate}
    \item \textcolor[RGB]{147,30,116}{magistrale procesor - memorie}
    \begin{itemize}
        \item scurte, rapide
        \item de obicei sincrone (sincronizare pe frecvenţa de tact a procesorului)
        \item monopol de firmă producătoare
    \end{itemize}
    \item \textcolor[RGB]{147,30,116}{magistrale de intrare - ieșire (I/O Bus)}
    \begin{itemize}
        \item suficent de lungi, pot conecta mai multe tipuri de periferice
        \item se conectează la memorie prin 1. sau 3.
        \item respectă standarde de magistrală
    \end{itemize}
    \item \textcolor[RGB]{147,30,116}{magistrale de extensie (backplane)}
    \begin{itemize}
        \item permit adăugarea în sistem a diferite plăci interschimbabile prin inserarea acestora în conectorii de extensie ai magistralei (socluri sau sloturi)
        \item rezolvă atât conexiunea procesor – memorie cât şi conexiunea periferice – memorie
        \item respectă standarde de magistrală
    \end{itemize}
\end{enumerate}

\subsubsection{ \textcolor[RGB]{147,30,116}{Standardizarea magistralelor}}

     \hspace{0.5cm}$\rightarrow$ caracterul de sistem deschis al SC moderne
     
     $\rightarrow$ două clase importante de magistrale:
    \begin{enumerate}
        \item \textcolor[RGB]{147,30,116}{magistrale de sistem} – conectează UC la celelalte componente
        \item \textcolor[RGB]{147,30,116} {magistrale specializate} – optimizează conectarea şi comunicarea cu un anumit periferic
    \end{enumerate}


\begin{itemize}
    \item SCSI (Small Computer System Interface)
    \begin{itemize}
        \item specific pentru I/O Bus $\Rightarrow$ interfaţă fizică către o magistrală de extensie sau către o magistrală procesor - memorie
        \item controller SCSI – coordonează transferurile de la periferic spre memorie 
        
        $\rightarrow$FireWire (standard IEEE 1394)
        \begin{itemize}
            \item rapid, ieftin $\Rightarrow$ conectează componente audio-video
        \end{itemize}
    \end{itemize}
   \item ISA (Industrial Standard Architecture), EISA (Extended ISA)
   \begin{itemize}
       \item specific pentru magistrala de extensie pentru conectarea componentelor lente
   \end{itemize}
   \item PCI (Peripheral Component Interconnect)
   \begin{itemize}
       \item magistrale de mare viteză; specific pentru magistrala de extensie
       \item recunosc facilitatea Plug\&Play (autoconfigurare)
   \end{itemize}
   \item USB (Universal Serial Bus), WUSB (Wireless USB)
   \begin{itemize}
       \item interfaţă serială rapidă, bidirecţională
       \item prelucrarea paralelă
   \end{itemize}
   \item SATA (Serial Advanced Technology Attachment), eSATA (external SATA)
   \begin{itemize}
       \item foarte rapid, DAR cu alimentare externă (ex. HDD extern)
   \end{itemize}
   \item Bluetooth (standard IEEE 802.15)
   \begin{itemize}
       \item conectare wireless
   \end{itemize}
   \item DVI (Digital Visual Interface) vs HDMI (High-Definition Multimedia Interface)
\end{itemize}

\subsubsection {\textcolor[RGB]{147,30,116}{Comunicaţia serială - COM}}
\begin{itemize}
\item informaţia se transmite bit cu bit, \textcolor[RGB]{147,30,116}{pe o singură linie fizică, bi-direcţional}; în practică, pentru o comunicaţie serială sunt necesare minimum 3 fire: unul pentru transmisia într-un sens, altul pentru transmisia în celălalt sens şi al treilea pentru masă (pentru întoarcerea curentului);
\item standardul pentru comunicaţia serială este RS 232; uzual, porturile seriale de uz general sunt numite COM 1 şi COM 2
şi folosesc conectori de tip mufă/\textcolor[RGB]{147,30,116}{cuplă} cu 9 pini/\textcolor[RGB]{147,30,116}{orificii} de tip tată/\textcolor[RGB]{147,30,116}{mamă} (engl., male/\textcolor[RGB]{147,30,116}{female}), numit DE9;

\item \textcolor[RGB]{147,30,116}{tastatura} şi \textcolor[RGB]{147,30,116}{mouse}-ul sunt
conectate prin interfaţă serială;
\item pentru \textcolor[RGB]{147,30,116}{tastatură} se folosesc trei modele de conectori specifici:
\begin{itemize}
\item DIN – cu 5 pini
\item mini-DIN cu 6 pini, numit şi PS/2
\item USB


\item pentru \textcolor[RGB]{147,30,116}{mouse} se folosesc trei modele de conectori specifici:

\begin{itemize}
\item mini-DIN cu 6 pini, numit şi PS/2
\item DE9 de tip mamă al portului serial COM1, caz în care cablul mouse-ului este dotat cu un conector DE9 de tip tată
\item USB
\end{itemize}
\end{itemize}
\end{itemize}

\item PCI standard – Peripheral Component Interconnect

\begin{itemize}
\item este un sistem de interconectare paralel între un microprocesor și multiple dispozitive de înaltă performanță: grafică, full-motion video, rețele locale, etc;
\item implementat de obicei ca magistrală de expansiune pe 32 de biți, are capacitate de 64 de biți;
\item dezvoltat de Intel în 1993, \textcolor[RGB]{147,30,116}{PCI} acceptă funcționalitatea \textcolor[RGB]{147,30,116}{PnP} (Plug and Play sau Plug-n Play), avansând standardul \textcolor[RGB]{147,30,116}{ISA};
\item \textcolor[RGB]{147,30,116}{PCI} a fost mult mai rapid decât \textcolor[RGB]{147,30,116}{ISA} și treptat
a înlocuit \textcolor[RGB]{147,30,116}{ISA} pe piață;
\item \textcolor[RGB]{147,30,116}{PC-urie / PCI} exprimă o conexiune în serie care funcționează mai mult ca o rețea decât ca o magistrală; în loc de un magistrala care gestionează date din mai multe surse, \textcolor[RGB]{147,30,116}{PCI-urile} au un \textcolor[RGB]{147,30,116}{comutator} care controlează mai multe conexiuni în serie punct cu punct .
\end{itemize}

\item AGP standard – Accelerated Graphics Port

\begin{itemize}
\item cele mai comune interfețe de magistrală între adaptorul video și Northbridge sunt standardele \textcolor[RGB]{147,30,116}{AGP} bazate pe standardele \textcolor[RGB]{147,30,116}{PCI} timpurii;
\item a fost dezvoltat în 1996 de Intel ca a paralelă de mare viteză punct la punct canal lat de 32 de biți pentru atașarea
\textcolor[RGB]{147,30,116}{plăcii video/grafice} și \textcolor[RGB]{147,30,116}{acceleratoare 3D} pentru un sistem informatic, în primul rând pentru a permite controlerelor grafice
 acces direct la sistemul de memorie (ajută la accelerarea pe
\textcolor[RGB]{147,30,116}{grafica computerului 3D});
\item în prezent, PCI-ul pe \textcolor[RGB]{147,30,116}{16 biți expres} a fost înlocuit de magistrala grafica \textcolor[RGB]{147,30,116}{AGP}
\end{itemize}

\subsubsection{\textcolor[RGB]{147,30,116}{Posibilităţi de a conecta diferitele
echipamente de intrare-ieşire la un PC}}

In evoluţia SC:
\begin{itemize}
\item există destul de multe posibilităţi de a conecta diferitele
echipamente de intrare-ieşire la un PC;
unele dintre acestea necesită:
\begin{itemize}
\item deconectarea SC de la reţeaua electrică;
\item desfacerea box-ului calculatorului şi introducerea unei plăci de extensie într-unul din conectorii PCI sau ISA;
\item dacă plăcile nu sunt de tip PnP, acestea au comutatoare sau jumpere care trebuie configurate;
\item apoi trebuie reconectat calculatorul;
\end{itemize}
\item altele pot fi conectate la una din cuplele magistralelor standard (paralelă sau serială), însă numărul acestora este limitat şi, în plus, nu asigură o viteză de transfer suficientă;
\item există nenumărate tipuri de conectori şi, respectiv, cabluri.
\end{itemize}

\textcolor[RGB]{124, 13, 208} {Există mai multe metode de conectare la rețeaua de internet, precum cabluri metalice și de fibră optică, chiar submarine, legături fără fir prin unde radio cum ar fi Wi-Fi, WiMAX sau Bluetooth, prin raze infraroșii, ca de exemplu IrDA, sau prin intermediul sateliților.}

O metode de conectare la rețea este conectorul RJ45 (Registered Jack 45), un conector cu 8 fire, folosit în reţele locale, în special de tip Ethernet. \textbf{ \textcolor[RGB]{147,30,116}{Arată la fel ca RJ11}} \textbf{folosit în telefonie}, doar că este mai lat.

\subsubsection{\textcolor[RGB]{147,30,116}{Magistrala USB}}

\begin{itemize}
\item a fost introdusă pentru a înlătura toate neajunsurile anterioare.
\item Obiectivele acestei magistrale au fost:
\begin{enumerate}
\item utilizatorii nu trebuie să aibă de poziţionat comutatoare sau jumpere
pe plăci sau pe echipamente;
\item pentru a instala noi dispozitive de I/O, utilizatorii nu trebuie să
deschidă box-ul calculatorului;
\item trebuie să existe un singur tip de cablu, pentru conectarea tuturor
dispozitivelor;
\item ataşarea a până la 127 de dispozitive la un calculator;
\item dispozitivele trebuie să poată fi instalate în timp ce calculatorul
funcţionează*;
\item după instalarea unui nou dispozitiv să nu trebuiască să fie repornit
calculatorul;
\end{enumerate}
\end{itemize}

\section{\fontsize{12}{16}\selectfont UNITATEA DE MEMORIE}
{\color{black} \rule{\linewidth}{0.5mm}}
%\section{Introducere în sistemele de calcul}


%\large{\textbf {Cuprins}}
\begin{tcolorbox}[colframe=black, colback=white]
\begin{enumerate}
\item Structura fizică a memoriei

\item Clasificarea dispozitivelor de memorare

\item Tehnologii de realizare a memoriilor. Memorii semiconductoare

\item Structura ierarhică de organizare a memoriei

\end{enumerate}
\end{tcolorbox}

\textcolor[RGB]{124, 13, 208}{Unitatea de memorie este unitatea funcțională a calulatorului, destinată păstrării conținutului - date, informații, instrucțiuni - pentru toate tipurile de beneficiari - umani, virtuali (aplicații, cloud).}
\\
 \textcolor[RGB]{147,30,116}{Unitatea de memorie }  este o colecție de:
 \begin{itemize}
     \item  \textcolor[RGB]{147,30,116}{medii de memorare} = componente destinate memorării datelor;
     \item  \textcolor[RGB]{147,30,116}{soluții electrice/magnetice}, necesare pentru:
     \begin{itemize}
         \item gestiunea memoriei;
         \item transferul de date/instrucțiuni înspre și dinspre memorie;
     \end{itemize}
 \end{itemize}
 \end{itemize}

 \subsection{Structura fizică a memoriei}

\begin{center}
    \textbf{\textcolor[RGB]{147,30,116}{memoria = mediu de memorare + circuite}}
\end{center}

\subsubsection{\textcolor[RGB]{147,30,116}{Cuvinte cheie:}}
\begin{itemize}
    \item bit (\textcolor[RGB]{124, 13, 208}{valoare binară - binary digit}), bit de informație (\textcolor[RGB]{124, 13, 208}{bitul împreună cu contextul lui, cu toate informațiile care vin împreună cu detaliul de valoare})
    \item celulă-bit 
    \begin{itemize}
        \item particule magnetice (\textcolor[RGB]{124, 13, 208}{structura care rezolvă memorarea pe componentele de tip disc magnetic}) cu polaritate (\textcolor[RGB]{124, 13, 208}{0 sau 1 logic}) specifică $\rightarrow$ discuri magnetice
        \item porți logice NAND/NOR $\rightarrow$ memorii semiconductoare
        \begin{itemize}
            \item circuit electronic cu două stări stabile - bistabil, condensator $\rightarrow$ RAM, registre
            \item dispozitiv electronic cu două stări - diodă, tranzistor $\rightarrow$ ROM, flash, SSD
        \end{itemize}
    \end{itemize}
    \item locație de memorie (\textcolor[RGB]{124, 13, 208}{suport de memorare, care poate să aibă în fiecare moment un conținut specific}), octet, cuvânt de memorie (capacitatea cuvântului de memorie este dată de lărgimea de bandă a magistralei-procesor (numărul de conductori))
    \item adresă de memorie (octet adresabil - \textcolor[RGB]{124, 13, 208}{fiecare octet în memorie este identificat in mod unic prin adresarea octetului respectiv})
    \item capacitatea memoriei, byte (\textcolor[RGB]{124, 13, 208}{succesiune de 8 biți})
    \begin{itemize}
        \item 1KB, 1MB, 1GB, 1TB - multipli informatici (puteri ale lui 2)
    \end{itemize}
    \item LSB - Least Significant Bit - \textcolor[RGB]{124, 13, 208}{cel mai puțin semnificant bit la dreapta cu numerotarea puterii bazei 2 de la dreapta la stânga}
    \item MSB - Most Significant Bit - \textcolor[RGB]{124, 13, 208}{cel mai semnificant bit la dreapta cu numerotarea puterii bazei 2 de la stânga la dreapta crescător}
\end{itemize}

\subsubsection{\textcolor[RGB]{147,30,116}{Celula binară} (celula de memorie binară)}
\begin{itemize}
    \item \textcolor[RGB]{147,30,116}{puncte magnetice pe platane dure} care se rotesc la viteze mari pentru a permite axului cu capete de scriere-citire să acceseze memoria sau să scrie în memorie (prin stabilirea polarității magnetice corespunzătoare a punctelor respective) $\rightarrow$ \textcolor[RGB]{147,30,116}{disc magnetic / disc dur}
    \item \textcolor[RGB]{147,30,116}{puncte de pe disc care reflectă lumina și care au polaritate diferită}, în funcție de orientarea câmpului magnetic $\rightarrow$ \textcolor[RGB]{147,30,116}{disc magneto-optic (disc magnetic și tehnologie optică de memorare}
    \item \textcolor[RGB]{147,30,116}{puncte pe disc cu iluminare a suprafeței cu unde laser și observare a reflexiei} $\rightarrow$ \textcolor[RGB]{147,30,116}{disc optic}
    \item porți logice \textcolor[RGB]{147,30,116}{NAND / NOR} $\rightarrow$ \textcolor[RGB]{147,30,116}{memorie semiconductoare}
    \begin{itemize}
        \item dispozitive semiconductoare - diode $\rightarrow$ \textcolor[RGB]{147,30,116}{bloc ROM}
        \item dispozitive semiconductoare - tranzistori $\rightarrow$ \textcolor[RGB]{147,30,116}{NAND flash memory: stick, card, SSD}
        \item bistabili / circuite bistabile (latches, flip-flops) $\rightarrow$ \textcolor[RGB]{147,30,116}{bloc SRAM}
        \item condensator $\rightarrow$ \textcolor[RGB]{147,30,116}{bloc DRAM}
        
        \end{itemize}
\end{itemize}

\begin{center}
    \textcolor[RGB]{147,30,116}{Memoria nu este altceva decât o stare care persistă în timp.\\Celula bit ține minte un bit de informație la un moment dat, în anumite condiții.}
\end{center}
 
 \subsection{Clasificarea dispozitivelor de memorare}
 Principalele \textcolor[RGB]{147,30,116}{tehnologii de realizare a memoriilor}: tambur magnetic, inele de ferită, cartele perforate, bandă magnetică, disc magnetic, circuite semiconductoare, procedee optice.
 \\ După \textcolor[RGB]{147,30,116}{ tipul de acces la date}:
 \begin{itemize}
     \item \textcolor[RGB]{147,30,116}{memorii cu acces aleator} - ex. RAM, ROM, Flash
     \begin{itemize}
         \item locațiile pot fi citite/scrise pe bază de adresă într-o ordine oarecare
         \item fiecare locație poate fi selectată aleator și accesată direct
     \end{itemize}
     \item \textcolor[RGB]{147,30,116}{memorii cu acces asociativ} - ex. memorie Cache
     \begin{itemize}
         \item locația căutată este identificată prin conținut
     \end{itemize}
     \item \textcolor[RGB]{147,30,116}{memorie cu acces secvențial} - ex. benzi magnetice
     \item \textcolor[RGB]{147,30,116}{memorie cu acces direct} - ex. disc
 \end{itemize}

\subsection{Memorii semiconductoare}

\subsubsection{\textcolor[RGB]{147,30,116}{Definiție}}
\textcolor[RGB]{147,30,116}{Memorii semiconductoare} = circuite integrate bazate pe metale semiconductoare (Si, Ge);
\\
\textcolor[RGB]{147,30,116}{Celulă-bit} = circuit de memorie cu capacitatea de 1 bit;
\\
\textcolor[RGB]{147,30,116}{Cuânt-memorie} = conținutul unui cuvânt de memorie;
\\ 
\textcolor[RGB]{147,30,116}{Spațiu de adresare} = domeniul adreselor locațiilor unui circuit de memorie;

\subsubsection{\textcolor[RGB]{147,30,116}{Parametrii caracteristici}}
\begin{itemize}
    \item \textcolor[RGB]{147,30,116}{geometria}: dispunerea celulelor-bit, lungimea cuvântului-memorie, modul de adresare a cuvintelor;
    \item \textcolor[RGB]{147,30,116}{capacitatea};
    \item \textcolor[RGB]{147,30,116}{latența} sau \textcolor[RGB]{147,30,116}{timpul de acces}: t$_{acces}$ = t$_{DateOut}$ - t$_{AdrIn}$;
    \item \textcolor[RGB]{147,30,116}{ciclul memoriei}: intervalul de timp dintre două aplicări succesive de adresă pe liniile de intrare în circuitul de memorie; se măsoară în ns sau $\mu$s;
    \item \textcolor[RGB]{147,30,116}{puterea consumată}: se măsoară în $\mu$W/bit;
    \item \textcolor[RGB]{147,30,116}{volatilitatea}: memoria volatilă VS memoria nevolatilă (permanentă);
    \item \textcolor[RGB]{147,30,116}{tehnologia de realizare}:
    \begin{itemize}
        \item \textcolor[RGB]{147,30,116}{memorii bipolare}: foarte rapide, densitate mică de integrare, tehnologii: TTL, ECL;
        \item \textcolor[RGB]{147,30,116}{memorii MOS}: latență mai mare, densitate mare de integrae, putere consumată mică, viteză mică la operații de scriere-citire; ex. \textcolor[RGB]{147,30,116}{memoriile flash} = carduri de memorie, dispozitive flash USB, camere digitale, telefoane mobile
    \end{itemize}
\end{itemize}

\subsubsection{\textcolor[RGB]{147,30,116}{Operații specifice}}
\begin{itemize}
    \item citire
    \begin{itemize}
        \item \textcolor[RGB]{147,30,116}{ROM} (Read Only Memory) = memorii de capacitate mică, cu viteză mare de acces la date; actual - componentă UCP pentru programul de control sau ca suport pentru BIOS (Basic Input Output Sysyem = set de aplicații low-level pentru: inițializarea sistemului, verificarea inițială a perifericelor, accesul primar la periferice, funcțiile de bază pentru unele dispozitive auxiliare (dispozitive mobile, controlere de rețea, controlere video));
    \end{itemize}
    \item citire și scriere - tehnologie pentru citire-scriere
    \begin{itemize}
     \item \textcolor[RGB]{147,30,116}{RAM} (Random Acces Memory) = memorii cu acces aleator
        \item \textcolor[RGB]{147,30,116}{SRAM} (Static RAM) - usual, celula-bit a unei memorii RAM statice este un bistabil;
        \item \textcolor[RGB]{147,30,116}{DRAM} (Dynamic RAM) - uzual, celula-bit a unei memorii RAM dinamice este un condensator;
    \end{itemize}
   \item programare - memorii PROM
   \item regenerare - citire incompletă
   \begin{itemize}
       \item utilă în memoriile DRAM;
       \item rata uzuală de regenerare este ~2ms/NN, unde NN = numărul de nivele de celule - bit din geometria memoriei (ex. 2ms/128 $\approx$ 15,625$\mu$s)
   \end{itemize}
\end{itemize}

\subsection{Structura ierarhică de organizare}

\begin{itemize}
\item În timpul execuţiei lor, programele accesează în orice
moment o zonă relativ mică a spaţiului de adrese.
\item Această observaţie a condus la delimitarea unor aspecte fundamentale în utilizarea memoriei, care sunt recunoscute ca \textcolor[RGB]{147,30,116}{principiul localizării}:
\begin{itemize}
\item \textcolor[RGB]{147,30,116}{principiul localizării temporale}
\item \textcolor[RGB]{147,30,116}{principiul localizării spaţiale}
\end{itemize}
\item Aplicarea principiului localizării a condus la organizare ierarhică a memoriei. O \textcolor[RGB]{147,30,116}{ierarhie de memorie} constă în nivele de memorii cu viteze de acces diferite şi de dimensiuni diferite
\end{itemize}

\begin{itemize}
\item p.v. \textcolor[RGB]{147,30,116}{\textbf{utilizator}}
\begin{itemize}
\item \textcolor[RGB]{147,30,116}{memorie externă}
\begin{itemize}
\item este la dispoziţia \textcolor[RGB]{147,30,116}{utilizatorului}
\item se conectează la SC prin \textcolor[RGB]{147,30,116}{interfeţe I/O}
\end{itemize}
\item \textcolor[RGB]{147,30,116}{memorie internă}
\begin{itemize}
\item este accesibilă pentru \textcolor[RGB]{147,30,116}{UCP}
\item este accesată direct de \textcolor[RGB]{147,30,116}{UCP} prin intermediul \textcolor[RGB]{147,30,116}{magistralelor}, fără 
intermediul canalelor de comunicaţie de I/O
\end{itemize}
\end{itemize}
\item p.v. \textcolor[RGB]{147,30,116}{\textbf{programator}} – informaţia stocată în memorie trebuie corelată cu
programul curent de executat:
\begin{itemize}
\item \textcolor[RGB]{147,30,116}{memorie secundară}
conţine informaţia \textcolor[RGB]{147,30,116}{păstrată} pentru o \textbf{eventuală} utilizare ulterioară
\item \textcolor[RGB]{147,30,116}{memorie principală}
conţine informaţia \textcolor[RGB]{147,30,116}{în curs de} utilizare (utilizată \textcolor[RGB]{147,30,116}{curent} de UCP)
\end{itemize}

\item ierarhie de memorie

\begin{itemize}
\item constă din \textbf{nivele de memorie}, cu diferite valori pentru \textcolor[RGB]{147,30,116}{timpul de acces} și \textcolor[RGB]{147,30,116}{capacitate}; cu cât crește \textbf{distanța} de la procesor la memorie, cu atât mai mult cresc și capacitatea componentelor de memorare și timpul de acces la memoriile respective;
\item se referă la utilizarea eficientă a nivelelor de memorie dispuse de tip mai mare, mai lent (larger but slower) - \textcolor[RGB]{147,30,116}{high storage capacity with low average latency};
\item prin faptul că \textbf{încarcă} și \textbf{memorează} conținut ȘI \textbf{din} locațiile \textbf{vecine}
celor accesate, \textcolor[RGB]{147,30,116}{memoria cache} este foarte probabil să conțină mereu
datele solicitate de procesor; dacă \textcolor[RGB]{147,30,116}{datele necesare procesorului} nu sunt
găsite în cache, ele \textbf{trebuie} aduse de pe \textbf{nivelele următoare de memorie} (mem principală, mem secundară, mem de arhivare) – \textcolor[RGB]{147,30,116}{datele sunt copiate numai între două nivele adiacente la un moment dat}.

\end{itemize}
\end{itemize}

\section {\fontsize{12}{16}\selectfont {FUN FACTS}}
{\color{black} \rule{\linewidth}{0.5mm}}
\begin{itemize}

\item \textcolor[RGB]{124, 13, 208}{Acum peste 2000 de ani, s-a realizat probabil primul calculator, mecanismul Antikythera, datând din anul 87 î.e.n. și folosit aparent pentru calcularea mișcărilor planetelor. Tehnologia care a stat la baza acestui mecanism nu este cunoscută.}
\item \textcolor[RGB]{124, 13, 208} {Ada este un limbaj de programare folosit în Departamentul apărării din SUA. Acest limbaj a fost creat în 1980 în memoria fiicei poetului englez George Byron, Ada Byron, care în 1842-1843 a scris primul algoritm de calcul, aceasta fiind considerată primul programator.}
\item  \textcolor[RGB]{124, 13, 208}{Alan Turing, considerat a fi părintele informaticii și inteligenței artificiale teoretice, a creat primul program de jucat șah pentru calculator. Cum la acel moment nu existau calculatoare pentru a fi testat, acest lucru a fost făcut cu ajutorul creionului si al hârtiei.}
\item  \textcolor[RGB]{124, 13, 208}{Principiile lui John von Neumann sunt incluse în fiecare calculator modern, tabletă sau telefon. De asemenea, acesta a fost și unul dintre primii susținători ai inteligenței artificiale, propunând ideea mașinilor cu auto-replicare. }

\end{itemize}
